# 橙攝器材價目表：個人 Android 查價 Web App

## 1. 目標

建立一個以本人使用為主、可安裝到 Samsung S23+ 桌面的手機版網頁 App。它專注於快速查閱橙攝公開的器材價目與品項詳情；不處理登入、預約、付款或庫存判定。

每個器材項目都會收錄其公開的完整詳情文字與價目欄位，並保留「開啟官方頁面」連結，讓最新公告與實際可租狀態仍回到原始網站確認。

## 2. 手機上會怎麼使用

1. 第一次用 Chrome 開啟本 App 的網址，選擇「安裝應用程式」或「加到主畫面」。
2. 桌面會出現獨立圖示；日後點圖示即可全螢幕開啟，不必再找書籤。
3. 可直接搜尋名稱、型號或品牌，也可用類別和每日租金篩選。
4. 點選項目可在 App 內看完整公開規格／說明與價目欄位；要進行預約、登入或查看網站其他內容時，按「前往橙攝官方頁面」。
5. App 開啟且有網路時，會檢查是否有新版資料；發現後自動下載。另保留「立即更新」按鈕與資料更新時間。
6. 沒網路時，仍可查閱最近一次成功下載的價目資料。

## 3. 每日更新的實際運作

```
橙攝公開價目頁
      │ 每天一次（固定低頻率）
      ▼
資料更新程式：讀取分類與品項、檢查格式、產生資料檔
      │ 成功才發布；失敗保留前一份可用資料
      ▼
安全 HTTPS 網址上的 Web App
      │ App 每次開啟時檢查版本
      ▼
你的 Samsung S23+ 已安裝 App
```

手機本身不需要一直開著，也不需要每天重新安裝。更新工作會在網路端的排程服務執行；手機只在你打開 App 時取得已準備好的新版資料。

### 更新失敗時的處理

- 不會把空白或半套資料覆蓋到手機；會繼續使用上一份完整資料。
- App 會顯示「資料最後成功更新時間」和更新失敗提示。
- 會提供一個管理用的手動更新入口，作為排程失敗時的備援。
- 價格、可租狀態與預約結果仍以官方網站為準。

## 4. 建議的畫面與功能範圍

### 首頁（緊湊、單手操作）

- 固定在頂端的搜尋框。
- 常用分類橫向按鈕：機身、鏡頭、錄影、收音、燈光、穩定器、週邊等。
- 依每日租金排序或篩選。
- 最近查看與我的收藏。

### 器材清單卡片

- 品名／型號（兩行內顯示）。
- 每日租金作為最醒目的資訊。
- 押金與本票金額以緊湊次要文字顯示。
- 所屬分類、最後更新時間、官方頁面連結。

### 器材詳情

- 完整名稱、公開規格／說明、全部價目欄位與原始類別。
- 可選擇加入收藏。
- 「在官方網站開啟」按鈕；一律連回原網址，不複製預約流程。

### 第一版不納入的功能

- 官方帳號登入、預約、結帳、會員資料。
- 自行判定庫存、保留器材或即時可租狀態。
- 大量下載或重製官方產品圖片與品牌素材。

## 5. 技術與部署建議

採用「靜態 PWA + GitHub Pages + 每日資料更新」：手機 App 本身很輕、開啟快，可快取離線資料；更新程式以 GitHub Actions 在雲端每天執行一次，將整理好的 JSON 資料發布到 GitHub Pages。

GitHub Pages 是公開 HTTPS 網站；不可將「知道網址才可開啟」視為存取保護。第一版不放任何帳號、個資、預約資料或私密筆記，只保存原本已公開的文字、價格、來源網址與必要的圖片網址。若未來需要真正只限本人，改用具登入保護的服務。

### GitHub Pages 每日更新細節

1. GitHub Actions 每天在離整點較遠的固定時間開始（例如台灣時間 11:17）。
2. 更新程式依官方分類頁取得品項清單，再以低頻率讀取各品項的公開詳情頁。
3. 將品名、類別、每日租金、押金、本票金額、完整公開詳情、官方網址與擷取時間整理為資料檔。
4. 先檢查資料是否完整：分類數、品項數、必填欄位與網址；異常時不發布，保留上一版。
5. 成功後更新 `catalog.json` 與 `version.json`，並發布到 GitHub Pages。手機 App 下次開啟時讀取版本檔，只下載有變動的資料。
6. 手機不需要重新安裝；App 內也保留「檢查更新」按鈕。此按鈕只會取得已經發布的最新版，不能強迫官方網站立即重新擷取。

GitHub 排程可能在平台繁忙時延遲，因此「每日一次」是合理目標而非精確到分鐘的保證；資料頁會顯示最後成功更新時間。

### 為何不使用手機裡唯一的一個 HTML 檔

離線單一 HTML 檔很適合固定不變的計算機，但它的資料會停留在建立當天。手機瀏覽器不會自行改寫這個本機檔案；而直接從本機檔讀取官方網站又會受到跨網域與網站驗證限制。

因此採用兩層設計：安裝到手機的 App 介面仍是輕量靜態檔案，最新資料則放在 GitHub Pages。你照常從桌面開啟同一個圖示，App 會檢查並下載新版資料。這同時保有「像本機 App」的使用體驗與每日更新能力。

不建議採用「把官方網站整頁包進 App」或反向代理：手機排版問題不會真正改善，也會帶來登入、版權、快取與網站負載風險。

## 6. 資料與使用邊界

- 僅整理公開的價目表資訊，並標明來源為橙攝官方價目頁。
- 每個品項保留原始網址及資料擷取時間。
- 不儲存官方會員帳號、個資、預約資訊或付款資料。
- 以低頻率每日一次讀取；若網站明確禁止自動擷取或出現驗證機制，改採人工更新或先向店家取得同意。
- App 只作個人查價輔助，不宣稱為官方服務。

## 7. 分階段製作計畫

### 階段 A：確認資料範圍與介面（先做）

1. 盤點官方分類、品項欄位與連結形式。
2. 已確認：每一品項的公開詳情文字與價目欄位都在 App 內閱讀；租賃、登入和其他非價目頁只連回官方網站。
3. 以手機寬度製作可操作的介面原型，先用少量真實資料測試搜尋、篩選和緊湊閱讀。

驗收：在手機上能快速找到一項常租設備，並一鍵回到官方頁面。

### 階段 B：完整個人查價 App

1. 收錄所有公開分類與價目品項。
2. 完成搜尋、分類、排序、收藏、資料更新時間與官方連結。
3. 加入 PWA 安裝、離線快取、版本檢查與「立即更新」按鈕。
4. 在 Samsung S23+ 的 Chrome 實機測試安裝、搜尋、離線與更新。

驗收：可由桌面獨立開啟；網路中斷時仍看得到最近價目表；連結都正確開啟官方原頁。

### 階段 C：每日自動更新與維護

1. 建置每日一次的更新排程。
2. 先比對新舊資料再發布，記錄新增、刪除、價格異動與錯誤。
3. 設定失敗時保留舊資料與簡單通知／日誌。
4. 實測連續多日更新，確認網站驗證或頁面格式變動時不會破壞 App。

驗收：資料檔每日更新且 App 下次開啟會取得新版；異常時可知道最後成功資料的時間。

## 8. 需要你在開始製作前確認的兩件事

1. 已確認：第一版收錄公開品項詳情、品名、每日租金、押金、本票金額、分類與官方連結；非價目流程回原站。
2. 已確認：採 GitHub Pages。此服務是公開 HTTPS 網頁，不放任何非公開資訊；若日後要限制本人存取，再更換有登入保護的託管方式。

## 9. 專案檔案規則

- 本專案所有程式、資料樣本、更新紀錄、測試截圖、使用說明，均存放在本資料夾內。
- 不在手機中保存唯一資料來源；任何更新都可由此專案資料夾重建。
- 後續建立子資料夾時，預定使用 `app/`、`data/`、`scripts/`、`docs/`、`tests/` 與 `logs/`。
